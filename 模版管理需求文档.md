9.4.2.2 主体配置

##### 数据行维度

一条配置代表「指定利润中心」在「指定税率条件」和「指定金额类型」场景下使用「指定模版」开具发票。

- **维度1：利润中心编码**（必填）——系统已维护的利润中心，如 `200000`
- **维度2：税率条件**（必填）——支持单税率（如 `5%`）、多税率（如 `5%,13%`）或通配值 `全部`
- **维度3：金额类型**（必填）——枚举值：`正数`、`负数`、`全部`
- **维度4：适用模版**（必填）——系统中已启用且存在的模版编码，如 `XSFPMB000001`

**唯一键**：在 (利润中心编码, 税率条件, 金额类型) 三维度上不可重复，且需遵守「全部」通配互斥规则。

支持以下两种新增方式：
- **页面新增**：在主体配置 Tab 单条录入
- **导入新增**：下载「主体配置导入模版.xlsx」，按列补全后上传导入

##### 页面查询条件

主体配置 Tab 列表上方提供查询条件栏，采用「标签：下拉框」格式排列，支持以下两项筛选，均为**多选**方式，需点击「查询」按钮触发检索：

| 查询条件 | 输入方式 | 展示值 | 查询能力 | 说明 |
|:---|:---|:---|:---|:---|
| **利润中心编码** | 下拉多选框 | `编码 - 名称`，如 `200000 - 加拿大` | 支持通过**编码**或**公司名称**进行模糊匹配 | 下拉选项来源为系统已维护的利润中心列表；选中后以标签形式展示在输入框内，可逐条移除 |
| **适用模版编码** | 下拉多选框 | `模版编码 - 模版名称`，如 `XSFPMB000001 - FBU-加拿大-5%GST` | 支持通过**模版编码**或**模版名称**进行模糊匹配 | 下拉选项来源为系统中已录入的模版列表；无选中项时展示占位文案「请选择」 |

**交互规则**
- 布局样式：查询条件按「利润中心编码：下拉框」「适用模版编码：下拉框」横向排列，右侧紧跟「查询」和「重置」按钮。
- 多选标签：选中项以胶囊标签展示在输入框内，点击标签右侧 × 可移除该选中项，点击输入框展开下拉面板。
- 模糊查询：在下拉面板顶部的搜索框输入关键词，实时过滤下拉选项；清空搜索框后恢复展示全部选项。
- 查询触发：点击「查询」按钮后，系统按已选条件过滤列表；无匹配数据时展示空态提示。
- 重置按钮：点击「重置」清空所有已选查询条件，列表恢复展示全部数据。
- 默认状态：进入页面时两个查询条件均为空，列表展示全部配置记录。

##### 主体配置导入新增 — 校验规则总览

导入采用**原子性校验**（All-or-Nothing）：系统一次性扫描导入文件的**全部数据行**并与系统存量比对，只要存在任意一行校验不通过，即整批数据拒绝导入，不写入任何数据。错误以「逐行清单」形式一次性汇总反馈。

###### 校验链（五步法）

系统按以下顺序逐行执行校验；某行可能在多个校验项上同时报错，全部记录后统一展示。

| 步骤 | 校验类别 | 校验内容 | 失败等级 |
|:---|:---|:---|:---:|
| 步骤一 | 格式预检 | 列数=4；利润中心编码、税率条件、金额类型、适用模版均不可为空 | 阻断 |
| 步骤二 | 值域校验 | 利润中心编码、适用模版编码、金额类型值是否在系统已定义 | 阻断 |
| 步骤三 | 格式兼容 | 模版编码忽略大小写；多税率分隔符支持英文逗号 `,`、中文逗号 `，`、空格 | 阻断 |
| 步骤四 | 唯一性校验 | (利润中心编码, 税率条件, 金额类型) 在系统内不重复，且不违反「全部」通配互斥 | 阻断 |
| 步骤五 | 模版状态校验 | 适用模版必须在系统中存在且处于「启用」状态 | 阻断 |

以下逐项展开规则与错误提示语。

###### 步骤一：格式预检

| 场景 | 判定规则 | 错误提示语 |
|:---|:---|:---|
| 列数不等于4 | 按英文逗号拆分后列数≠4，或存在未正确闭合的引号 | 「第 {行号} 行列数不正确，主体配置导入模版必须包含 4 列：利润中心编码、税率条件、金额类型、适用模版」 |
| 利润中心编码为空 | 去除首尾空白后为空 | 「第 {行号} 行利润中心编码为空，请补全后重试」 |
| 税率条件为空 | 去除首尾空白后为空 | 「第 {行号} 行税率条件为空，请补全后重试」 |
| 金额类型为空 | 去除首尾空白后为空 | 「第 {行号} 行金额类型为空，请补全后重试」 |
| 适用模版为空 | 去除首尾空白后为空 | 「第 {行号} 行适用模版编码为空，请补全后重试」 |

> **说明**：若文件包含表头行，表头行计入行号统计；导入模版第1行固定为表头，实际数据从第2行起。

###### 步骤二：值域校验

| 校验项 | 判定规则 | 错误提示语 |
|:---|:---|:---|
| 利润中心编码不存在 | 该编码不在系统已启用的利润中心列表中 | 「第 {行号} 行利润中心编码「{实际值}」在系统中不存在，请确认后重试」 |
| 金额类型值非法 | 值不在允许集合 `正数 / 负数 / 全部` 中 | 「第 {行号} 行金额类型「{实际值}」不在允许范围内（正数 / 负数 / 全部），请修正后重试」 |
| 适用模版编码不存在 | 该编码不在系统【模版管理】已录入的模版列表中 | 「第 {行号} 行适用模版编码「{实际值}」在系统中不存在，请确认后重试」 |
| 适用模版已禁用 | 模版存在但状态为「禁用」 | 「第 {行号} 行适用模版「{实际值}」已被禁用，仅支持选择启用状态的模版，请修正后重试」 |

###### 步骤三：格式兼容

| 校验项 | 判定规则 | 处理说明 |
|:---|:---|:---|
| 模版编码大小写 | 不区分大小写匹配系统中已存在的模版编码 | 系统内部统一转大写后比对；`xsfPMB000001` 与 `XSFPMB000001` 视为同一编码 |
| 多税率分隔符兼容 | 税率条件中若包含多个税率，支持 `,`、`，`（全角逗号）或空格分隔 | 解析时统一替换为英文逗号后再拆分；例：`5%,13%`、`5%，13%`、`5% 13%` 均解析为 `['5%','13%']`；空格的兼容仅做解析容错，文档说明仍以英文逗号为准 |

###### 步骤四：唯一性校验（含「全部」通配互斥）

**配置唯一键**：以 (利润中心编码, **标准化后的**税率条件, 金额类型) 为唯一键。
- 税率条件为多税率时，先排序后拼接为标准化字符串再比对（例：`[13%,5%]` 标准化为 `13%,5%`）
- 行内自身重复的多税率也需去重后比对

**「全部」通配互斥规则**：

当税率条件或金额类型取值为 `全部` 时，视为**通配**，与该维度下的**所有具体值**互斥。互斥是**双向**的：

1. **税率条件 = "全部" 的互斥**
   - 若系统中已有某利润中心 + `全部` + 某金额类型的配置，则该利润中心 + `该金额类型` 下**不允许再新增任何具体税率**的配置。
   - 反之，若系统中已有某利润中心 + **具体税率** + 某金额类型的配置，则该利润中心 + `该金额类型` 下**不允许再新增税率 = "全部"** 的配置。

2. **金额类型 = "全部" 的互斥**
   - 若系统中已有某利润中心 + 某税率 + `全部` 金额类型的配置，则该利润中心 + `该税率` 下**不允许再新增任何具体金额类型**（正数/负数）的配置。
   - 反之，若系统中已有某利润中心 + 某税率 + **具体金额类型** 的配置，则该利润中心 + `该税率` 下**不允许再新增金额类型 = "全部"** 的配置。

3. **行内重复检测**
   - 导入文件内部，同一行如果两次出现相同配置（如重复上传同一行数据，或标准化后相同的重复行），视为内部重复。

| 冲突场景 | 错误提示语 |
|:---|:---|
| 导入行与系统存量重复（三维度完全一致） | 「第 {行号} 行：利润中心 {利润中心编码} + 税率 {税率条件} + 金额类型 {金额类型} 的配置在系统中已存在，不支持重复导入」 |
| 系统已有"全部"税率，导入具体税率（同利润中心+同金额类型） | 「第 {行号} 行：利润中心 {利润中心编码} 在「{金额类型}」场景下已配置税率通配（全部），不支持再新增具体税率配置」 |
| 系统已有具体税率，导入"全部"税率（同利润中心+同金额类型） | 「第 {行号} 行：利润中心 {利润中心编码} 在「{金额类型}」场景下已有具体税率配置，不支持新增税率通配（全部）」 |
| 系统已有"全部"金额类型，导入具体金额类型（同利润中心+同税率） | 「第 {行号} 行：利润中心 {利润中心编码} 在税率「{税率条件}」场景下已配置金额类型通配（全部），不支持再新增具体金额类型配置」 |
| 系统已有具体金额类型，导入"全部"金额类型（同利润中心+同税率） | 「第 {行号} 行：利润中心 {利润中心编码} 在税率「{税率条件}」场景下已有具体金额类型配置，不支持新增金额类型通配（全部）」 |
| 导入文件内部行重复 | 「第 {行号A} 行与第 {行号B} 行配置重复（利润中心{编码} + 税率{条件} + 金额类型{类型}），请去重后重试」 |

###### 步骤五：模版状态校验

| 场景 | 判定规则 | 错误提示语 |
|:---|:---|:---|
| 适用模版存在但已禁用 | 模版编码在数据库中存在，但 `status='disabled'` | 「第 {行号} 行适用模版「{编码}」当前为禁用状态，仅支持选择启用状态的模版，请修正后重试」 |
| 适用模版编码与系统大小写不一致（已在步骤三兼容，此处只做状态校验） | — | 已在步骤三处理 |

##### 校验失败反馈方式

###### 弹窗提示（第二步阻断型）

当导入文件校验不通过时，弹窗停留在**第一步（文件上传）**，在拖拽区下方以「错误清单面板」展示全部错误。

| 元素 | 说明 |
|:---|:---|
| 面板标题 | `导入校验未通过，共发现 {N} 条错误：` |
| 错误行列表 | 每行格式：「第 {行号} 行：{具体错误}」 |
| 示例文案 | 第 3 行：利润中心编码「999999」在系统中不存在；第 5 行：适用模版「XSFPMB000099」已被禁用；第 8 行：利润中心 200002 在「正数」场景下已配置税率通配（全部），不支持再新增具体税率配置 |
| 交互 | 用户可点击「重新选择文件」更换新文件；修正错误后重新上传 |

> **注意**：所有校验一次性执行完毕后统一反馈，不会遇到第一个错误就终止扫描。

###### 页面新增校验（与导入新增保持同等严格）

页面单条新增时，执行同样的五步校验逻辑：
- **保存前实时校验**：点击「保存」按钮时，一次性校验所有必填项与值域
- **唯一性校验**：同样遵循「全部」通配互斥规则，阻断保存并提示
- **提示方式**：表单下方红色文字提示；涉及唯一性互斥时，弹窗二次确认不通过，直接阻断

| 校验项 | 失败提示方式 |
|:---|:---|
| 利润中心编码未选择 | 表单字段下方红字提示「利润中心编码为必填项」 |
| 税率条件或金额类型为空 | 同字段级红字提示 |
| 适用模版不匹配（禁用/不存在） | 下拉框下方提示「仅支持选择已启用的模版」 |
| 唯一性冲突 | 弹窗提示「当前配置与系统中已存在的配置冲突：{详细说明}」 |

##### 主体配置匹配规则（开具时调用）

主体配置 Tab 中，系统按以下三维度匹配适用模版：

```
entity（利润中心编码） + tax_rate（税率条件） + sign_type（金额类型） → template_id
```

**匹配逻辑（严格匹配）**：
系统中同一个条件组合 `(利润中心编码, 税率条件, 金额类型)` **只能唯一匹配到一个模版**。不存在优先级降级或通配回退逻辑——只要该精确组合在主体配置中有记录，就返回对应模版；无记录则阻断开具。

> 说明：「全部」的通配含义在**配置录入阶段**通过互斥规则保证——若某利润中心 + 某金额类型下已配置税率为「全部」，则该利润中心 + 该金额类型下不允许再录入任何具体税率；反之亦然。金额类型维度同理。因此开具时仅需查找精确组合，无需通配降级。

「适用模版」下拉框仅展示与所选条件匹配且处于**启用状态**的模版。

**未匹配处理**：若主体在所选税率条件 + 金额类型下未配置任何适用模版（即下拉框无候选），保存时阻断并提示「该主体该场景未配置适用模版，请先在「主体配置」Tab 中新增配置后再来开具」。发票开具环节调用同一匹配逻辑，同样按此提示阻断，避免落库后才发现无模版可用。

##### 导入新增逻辑补充（原子性要点）

- 导入文件解析后，先完整加载到内存，**全量校验通过后再统一写入数据库**，不存在「部分成功」的场景。
- 若导入文件与系统存量完全一致（无新增数据），第二步展示「无变更」，允许用户确认导入（幂等操作，不报错）。
- 导入成功后，列表自动刷新，新配置按利润中心编码升序展示于对应分组下。

9.6 注意事项和说明
4. 模版更新不影响历史发票：模版导入更新后，旧版本自动归档，历史发票按归档版本回溯渲染，确保发票内容不因模版变更而变化。
5. 日期/金额格式列用法：「日期/金额格式」列仅作用于该字段最终值的呈现，对取值逻辑本身无影响。同一字段可独立指定格式（如金额字段填 #,##0.00、日期字段填 yyyy-MM-dd），无需关心 Excel 单元格样式。常见格式串示例：
  - 金额（千分位 + 两位小数）：#,##0.00
  - 金额（仅两位小数）：0.00
  - 金额（财务记账风格，负数加括号）：#,##0.00;(#,##0.00)
  - 带币种符号：¥#,##0.00 / HK$#,##0.00 / CA$#,##0.00
  - 日期（ISO）：yyyy-MM-dd
  - 日期（中文）：yyyy年MM月dd日
  - 月份：yyyy-MM
  - 百分比（如税率）：0.00%
  - 空值：留空表示不格式化，按原值输出
