### 功能说明

#### 模版导入更新

在「发票模版管理 → 模版管理」Tab 中，FSSC 发票管理员对单个形式发票模版进行**导入更新**操作：

- 支持上传单个 `.xlsx` 模版文件，文件内包含 **模版 Sheet**（渲染样式占位）与 **字段规则配置 Sheet**（红字字段 + 取值规则）两类工作表。
- 系统识别文件归属的模版编码后，**逐字段比对**该模版当前生效版本与新文件之间的差异，展示新增/更新/删除/跳过四类变更，由用户在第二步点击「确认导入」按钮后才生效。
- 导入成功后，新版本覆盖当前主表记录，旧版本自动归档至历史表，已生成的历史发票按归档版本回溯渲染，不受影响。
- 模版导入完成后，模版行末展示**规则解析状态**（全部配置化 / 含硬编码规则 N 条），便于管理员快速识别该模版的规则可维护性。

#### 字段规则自动解析（本期新增）

模版导入过程中，系统对**字段规则配置 Sheet** 执行**轻量解析器**，将可配置化的字段规则转为结构化配置持久化，供后端渲染引擎通用处理；超出轻量引擎范围的规则标记为需硬编码，由后端专用函数处理。

- **解析器**：按字段规则的「取值类型」分发到不同解析策略，输出统一的结构化 JSON。
- **配置化规则**（字段映射、SUM 聚合）：解析为结构化配置存入 `template_field_rule` 表。
- **硬编码规则**（条件分支、字符串拼接、流水号生成）：标记为不可配置，仅做只读展示，后端走专用函数处理。

##### 轻量解析器的含义

「轻量」是相对于通用表达式引擎而言的方案定位词，**不是指解析能力弱，而是指仅通过正则模式匹配识别规则，识别范围有限但实现简单、无外部依赖**。

| 方案形态 | 能力范围 | 本期是否采用 |
|:---|:---|:---:|
| **轻量解析器（本期）** | 仅正则匹配 3 种固定模式（字段映射、SUM 聚合、规则计算），其余一律归硬编码 | ✅ 是 |
| 中等表达式引擎 | 支持 IF/AND/OR/四则运算/字符串函数等通用表达式，类 Excel 公式 | ❌ 二期预留 |
| 图灵完备脚本 | 支持任意复杂逻辑（Python/Lua），灵活但安全风险高、维护成本大 | ❌ 不考虑 |

**轻量的具体表现**：解析器只识别以下 3 类取值类型，其他类型（如四则运算、IF 条件、文本函数、跨表查询等）一律归入硬编码，由后端以系统内置函数方式实现（如 `generate_invoice_date()`、`generate_invoice_no()`、`calculate_due_date()`）。

- 字段映射 → 正则匹配 `申请单→xxx` 或 `行项目→xxx` → 配置化
- 公式计算 → 正则匹配 `SUM(行项目.xxx)` → 配置化
- 规则计算 → 不匹配上述模式 → 硬编码（标记 `hardcode_reason`，后端走专用函数）

按当前摸底，**约 87% 的规则**可被本轻量解析器消化为结构化配置，**约 13%** 因涉及条件分支、流水号拼接、自定义日期格式等复杂逻辑，需走硬编码函数。这部分比例随各模版补齐「字段编码」列并规范化取值逻辑描述可进一步提升。

#### 范围与边界

- 字段规则的解析仅作用于「字段规则配置 Sheet」，不对模版 Sheet（渲染样式）做语义解析，样式差异由人工核对。
- 解析器的取值类型分发仅识别三类：字段映射、公式计算（SUM）、规则计算；其他类型（如四则运算、IF 条件、文本函数）一律归入硬编码。
- **样式归属**：
  - 文本对齐方式（居中 / 左对齐 / 右对齐）、字体、字号、颜色、边框、合并单元格等**静态视觉属性**均由**模版 Sheet 自身的单元格样式**定义，导入时原样保留至运行时渲染。
  - **金额与日期的呈现格式（千分位、小数位、日期格式串等）不归模版 Sheet**，而是由字段规则配置 Sheet 中的「日期/金额格式」列集中管控，运行时由后端按格式串显式格式化。理由：Excel 单元格样式难以集中维护，且运行时模板引擎透传不可控，集中到配置表更易维护、校验与审计。

### 原型说明

#### 模版列表行操作入口

模版列表每行操作列包含「**导入更新**」链接（与预览、下载、启用/禁用并列），点击后打开导入向导弹窗，弹窗标题动态显示「导入更新 - {模版编码}」。

#### 4 步导入向导弹窗

弹窗为单容器、四步骤、互斥显示的状态机，每次仅展示一个步骤。

| 步骤 | 名称 | UI 关键元素 | 状态切换 |
|:---:|:---|:---|:---|
| 第一步 | 文件上传 | 拖拽区 + 「选择文件」按钮 + 文件信息回显（文件名、大小） | 上传成功后自动进入第二步 |
| 第二步 | 变更差异确认 | 差异表（状态/编码/名称/备注）+ 底部统计摘要 | 用户点击「确认导入」进入第三步 |
| 第三步 | 导入进度 | 进度文本「正在导入... N%」+ 进度条 | 进度达到 100% 自动进入第四步 |
| 第四步 | 导入结果 | 结果图标 + 汇总文案 + 明细行 | 用户点击「完成」关闭弹窗 |

##### 弹窗底部按钮随步骤联动

| 步骤 | 取消按钮 | 确认按钮 | 完成按钮 |

| 第一步 | 显示，文案「取消」 | 隐藏 | 隐藏 |
| 第二步 | 显示，文案「取消」 | 显示，文案「确认导入」 | 隐藏 |
| 第三步 | 隐藏 | 隐藏 | 隐藏 |
| 第四步 | 隐藏 | 隐藏 | 显示，文案「完成」 |

##### 差异表行样式

| 变更状态 | 标识 | 文本 |
|:---:|:---:|:---|
| 新增 | 绿色「+ 新增」徽标 | `{编码} {名称} - 新模版` |
| 更新 | 黄色「~ 更新」徽标 | `{编码} {名称} - {差异说明}` |
| 跳过 | 灰色「- 跳过」徽标 | `{编码} {名称} - 无变化` |

#### 字段规则解析结果面板

在模版预览弹窗内，渲染效果下方增加「**字段规则解析结果**」面板，分两栏展示：

- 左栏「可配置化规则」：字段 key、字段编码、分类、取值类型、配置 JSON 摘要。
- 右栏「需硬编码规则」：字段 key、字段编码、分类、硬编码原因（取值类型=规则计算）。

模版列表行末同时以徽标展示「全部配置化」或「含硬编码规则 X 条」，点击可下钻至解析结果详情。

### 相关规则

#### 字段规则解析规则

##### 解析器输入：字段规则配置 Sheet

Sheet 每行包含以下列：

| 列名 | 说明 |
|:---|:---|
| 字段 key（红字内容） | Excel 模板中的红色字段标记 |
| 分类 | 申请单映射字段 / 系统生成/计算字段 / 行项目区字段 |
| 取值类型 | 字段映射 / 公式计算 / 规则计算 |
| 取值逻辑 | 自然语言描述的规则（解析器消费） |
| 说明 | 补充说明 |
| **字段编码（新增）** | 英文编码，解析器直接用作 `source_field` / `target_field` |
| **日期/金额格式（新增）** | 可选；Excel 自定义格式串（如 `#,##0.00` / `yyyy-MM-dd` / `0.00%`）；空值表示不格式化，原值输出 |

##### 解析器输出：`template_field_rule` 表

解析器按「取值类型」分发处理：

| 取值类型 | 解析策略 | 输出示例 | 是否可配置化 |
|:---|:---|:---|:---:|
| **字段映射** | 正则匹配「申请单→xxx」或「行项目→xxx」 | `{type:"field_map", source_scope:"apply", source_field:"buyer_name", value_format:"#,##0.00"}` | ✅ 是 |
| **公式计算** | 正则匹配 `SUM(行项目.xxx)` | `{type:"sum", target:"line_items", field:"amount", value_format:"#,##0.00"}` | ✅ 是 |
| **规则计算** | 复杂条件分支/拼接/流水号，不匹配上述模式 | 仅标记 `hardcode_reason`；如需格式化，由对应硬编码函数内部按内置格式输出 | ❌ 否（硬编码） |

> 注：`value_format` 字段为可选项，仅在字段规则配置 Sheet 中「日期/金额格式」列非空时写入。渲染时若 `value_format` 不为空，后端按格式串对最终值做格式化；为空则输出原值。格式串语法遵循 Excel 自定义格式规范（如 `#,##0.00` / `0.00%` / `yyyy-MM-dd` / `yyyy年MM月dd日` / `¥#,##0.00` / `#,##0.00;(#,##0.00)` 等），后端通过 ICU 或等价库解析。

##### 解析结果统计（示意）

| 分类 | 数量 | 可配置化 | 需硬编码 |
|:---|:---:|:---:|:---:|
| 申请单映射字段 | 6 条 | 4 条 | 2 条 |
| 系统生成/计算字段 | 4 条 | 4 条 | 0 条 |
| 行项目区字段 | 6 条 | 6 条 | 0 条 |
| **合计** | **16 条** | **14 条（87%）** | **2 条（13%）** |

> 注：随着各模版补充「字段编码」列并规范取值逻辑描述，可配置化比例可进一步提升。具体各模版的实际比例不在本节枚举。

#### 模版导入更新规则

1. **导入覆盖逻辑**：新模版导入后，旧版本自动归档到 `template_archive` 表，模版主表更新为最新版本。
2. **字段规则同步更新**：字段规则配置随模版导入同步更新，新增的规则插入，已存在的规则更新，`config_json` 重新解析生成。
3. **历史发票追溯**：模版更新后，已生成的历史发票仍按归档版本渲染，不受影响。
4. **解析状态展示**：模版行末的「规则解析状态」徽标随每次成功导入更新，枚举值、存储结构与边界处理见下一小节。

#### 规则解析状态徽标

##### 枚举值（3 个）

| 值 | 显示文本 | 徽标样式 | 触发条件 |
|:---:|:---|:---:|:---|
| `unparsed` | 未解析 | 灰色 badge-disabled | 模版从未导入，或导入失败后无规则数据 |
| `all_configured` | 全部配置化 | 绿色 badge-enabled | 可配置化条数 = 规则总条数（即硬编码条数 = 0） |
| `has_hardcode` | 含硬编码规则 N 条 | 黄色 badge-warn | 可配置化条数 < 规则总条数，N = 总条数 - 可配置化条数 |

##### 数据结构

存储在模版主表上（去正规化字段，避免每次列表查询都聚合 `template_field_rule`）：

| 字段 | 类型 | 说明 |
|:---|:---:|:---|
| `parse_status` | enum | `unparsed` / `all_configured` / `has_hardcode` |
| `rule_total_count` | int | 解析得到的字段规则总条数 |
| `rule_configured_count` | int | 其中可配置化条数 |
| `rule_hardcode_count` | int | 计算列 = total − configured，前端只展示 |

##### 更新逻辑

| 时机 | 行为 |
|:---|:---|
| 模版**首次导入成功** | 解析完成后按解析结果回填上述 4 个字段 |
| 模版**导入更新成功** | 同上，**覆盖**旧值 |
| 模版**导入失败** | **不更新**字段，保持上一次成功导入的状态（或维持 `unparsed`） |
| 模版**归档到 `template_archive`** | 主表字段保留（归档版本需保留快照用于历史发票追溯） |
| 模版**被删除** | 主表记录连带删除（归档表保留） |
| 列表查询 | 直接读主表字段，无需聚合 |

##### 边界情况

| 场景 | 处理 |
|:---|:---|
| 字段规则配置 Sheet 为空（0 条规则） | 视作 `all_configured`（0/0），徽标绿色「全部配置化」 |
| 解析器抛异常 | 回滚入库，状态维持上一次成功值；若从未成功过，保持 `unparsed` |
| 用户导入了一个**完全没有字段规则 Sheet**的文件 | 第二步校验阶段就阻断，不进入入库流程 |

#### 主体配置匹配规则

主体配置 Tab 中，系统按以下三维度匹配适用模版：

```
entity（主体编码） + tax_rate（税率条件） + sign_type（金额类型） → template_id
```

匹配优先级：**精确匹配 > 部分匹配**（未指定条件项视为通配）。

「适用模版」下拉框仅展示与所选条件匹配且处于**启用状态**的模版。

**未匹配处理**：若主体在所选税率条件 + 金额类型下未配置任何适用模版（即下拉框无候选），保存时阻断并提示「该主体该场景未配置适用模版，请先在「主体配置」Tab 中新增配置后再来开具」。发票开具环节调用同一匹配逻辑，同样按此提示阻断，避免落库后才发现无模版可用。

#### 校验逻辑

| 校验项 | 时机 | 失败处理 |
|:---|:---|:---|
| 文件后缀 | 第一步上传时 | 阻断上传，提示「仅支持 .xlsx 格式」 |
| 工作表完整性 | 第一步上传后解析 | 弹窗停留第一步，列出缺失工作表 |
| 模版编码一致性 | 第二步解析时 | 阻断导入，提示编码不匹配 |
| 「日期/金额格式」列格式串合法性 | 第二步解析时 | 阻断导入，提示「第 N 行「日期/金额格式」值「{value}」不是合法的 Excel 自定义格式串，请修正后重试」 |
| `config_json` 语法 | 第三步入库时 | 第四步明细列出失败规则，规则不写入 |

#### 失败场景与提示语

按导入流程的四个步骤逐一枚举失败场景、对应提示语与系统处理动作。

##### 第一步 文件上传

| 场景 | 提示语 | 处理 |
|:---|:---|:---|
| 文件后缀非 `.xlsx`（如 `.xls` / `.csv` / `.zip`） | 「仅支持 .xlsx 格式，请重新选择」 | 阻断上传，弹窗停留第一步 |
| 文件为空（0 字节） | 「文件为空，请重新选择有效文件」 | 阻断上传 |
| 文件无法解析 / 损坏（zip 解压失败） | 「文件解析失败，请确认文件未损坏后重试」 | 阻断上传 |
| 文件大小超限（> 10 MB，可配置） | 「文件过大（{实际大小} MB），请压缩后再上传（上限 10 MB）」 | 阻断上传 |
| 拖拽多个文件 | 「仅支持单文件导入，请只保留 1 个文件」 | 阻断上传，仅保留第一个 |
| 文件缺失「模版 Sheet」 | 「文件缺少「模版 Sheet」，无法导入」 | 阻断，弹窗停留第一步 |
| 文件缺失「字段规则配置 Sheet」 | 「文件缺少「字段规则配置 Sheet」，无法导入」 | 阻断，弹窗停留第一步 |

##### 第二步 变更差异确认

| 场景 | 提示语 | 处理 |
|:---|:---|:---|
| 模版编码不一致（文件编码 ≠ 当前操作编码） | 「文件中的模版编码为「{实际编码}」，与当前操作的「{当前编码}」不匹配，请重新选择文件」 | 阻断，停留在第一步 |
| 字段规则 Sheet 表头列不匹配 | 「字段规则 Sheet 列结构不符合规范，请参考模版说明后修正」 | 阻断，停留在第一步 |
| 「字段编码」列存在空值 | 「第 {行号} 行「字段编码」为空，请补齐后再导入」 | 阻断，停留在第一步，明细列出所有空行 |
| 字段 key 在同一 Sheet 内重复 | 「字段 key「{key}」在第 {行号A}、{行号B} 行重复出现，请去重后重试」 | 阻断，停留在第一步 |
| 「分类」字段值非法（不在三选一集合内） | 「第 {行号} 行「分类」值「{实际值}」不在允许范围内（申请单映射字段 / 系统生成/计算字段 / 行项目区字段）」 | 阻断，停留在第一步 |
| 「取值类型」字段值非法（不在三选一集合内） | 「第 {行号} 行「取值类型」值「{实际值}」不在允许范围内（字段映射 / 公式计算 / 规则计算）」 | 阻断，停留在第一步 |
| 字段规则 Sheet 数据行为 0（无任何规则） | 「字段规则配置 Sheet 无有效规则，请确认文件是否正确」 | 阻断，停留在第一步 |
| **模版 Sheet 中存在红字字段未在字段规则配置 Sheet 中定义**（孤儿红字，反向缺失） | 「模版中存在 {N} 个红字字段未配置取值规则：{key1, key2, ...}，请在字段规则配置 Sheet 中补齐后重新导入」 | 阻断，停留在第一步（避免渲染时空字段无法取值） |
| **字段规则配置 Sheet 中存在规则但模版 Sheet 中无对应红字字段**（孤儿规则，正向多余） | 「字段规则配置中存在 {N} 条规则在模版 Sheet 中无对应红字字段：{key1, key2, ...}，请删除多余规则或补充模版字段后重新导入」 | 阻断，停留在第一步（避免无 UI 承载的规则成为死配置） |
| 模版行无任何变化（全部跳过） | 「文件与当前版本完全一致，无需导入」 | 允许进入第三步（实质为幂等操作），第四步展示「无变更」 |

##### 第三步 导入执行

| 场景 | 提示语 | 处理 |
|:---|:---|:---|
| 数据库写入失败（连接断开 / 超时） | 「导入失败：数据库连接异常（错误码 {code}），请稍后重试」 | 整体回滚，进入第四步展示 |
| 旧版本归档失败（写入 `template_archive` 异常） | 「导入失败：归档历史版本失败，请联系运维」 | 整体回滚 |
| 解析器抛未知异常（正则匹配崩溃等） | 「导入失败：解析过程出现异常（错误码 {code}），请联系运维」 | 整体回滚，状态保持上一次成功值 |
| `config_json` 语法校验失败（单条规则） | 「第 {行号} 条规则「{field_code}」配置 JSON 非法，已跳过」 | **该条规则不写入**，其余规则继续入库，第四步明细列出 |
| 并发导入同一模版（两个管理员同时操作） | 「「{模版编码}」正在被「{另一操作人}」导入中，请稍后再试」 | 后端互斥锁，第二个请求直接拒绝，回到第一步 |
| 第三步期间用户强制刷新 / 关闭页面 | 「导入已提交，关闭页面不会中断。请等待列表自动刷新（约 30 秒），或联系运维核对」 | 弹窗外 toast 提示（如能感知）；后端事务继续执行 |

##### 第四步 结果展示

| 场景 | 提示语 | 处理 |
|:---|:---|:---|
| 全部成功 | 「导入成功，共写入 {N} 条规则」 | 模版主表更新，徽标按解析结果重算 |
| 部分成功 | 「导入完成：成功 {N} 条，跳过 {M} 条（详见明细）」 | 模版主表更新，明细区列出失败行号 + 原因 |
| 全部失败 | 「导入失败：所有规则均未写入，请修正后重试」 | 模版主表**不更新**，状态保持上一次成功值（或 `unparsed`） |
| 解析状态徽标重算 | — | 导入成功后按 `total / configured / hardcode` 重算并覆盖旧值 |

### 注意事项和说明

1. **字段编码列必须补齐**：各模版维护人员需在「字段规则配置」Sheet 中补填「字段编码」列，使用英文/英文点号命名（如 `buyer_name`、`line_item.amount`、`sum_line_item_amount`）。命名统一规范后，解析器才能正确映射，无需人工维护映射字典。

2. **取值逻辑描述规范化**：建议将取值逻辑统一为结构化短句，便于解析器正则匹配。例如：
   - 字段映射：`申请单→购买方名称`
   - SUM 聚合：`SUM(行项目.金额)`
   - 复杂规则保持自然语言，由硬编码函数处理。

3. **硬编码规则后端统一管理**：条件分支（ITS vs FPS）、流水号生成、字符串拼接+自定义日期格式等超出轻量引擎的规则，由后端以系统内置函数方式实现（如 `generate_invoice_date()`、`generate_invoice_no()`、`calculate_due_date()`），各模版共用同一套函数族，仅入参不同。

4. **模版更新不影响历史发票**：模版导入更新后，旧版本自动归档，历史发票按归档版本回溯渲染，确保发票内容不因模版变更而变化。

5. **二期扩展预留**：本期采用轻量解析器，覆盖约 87% 的字段规则。如果后续业务规则频繁调整，可将条件分支、字符串拼接能力升级为中等表达式引擎，届时硬编码规则可平滑迁移到配置化，数据库表结构和解析器框架无需推倒重来。

6. **解析状态的可读性**：模版列表行末的「规则解析状态」徽标为**只读信息**，仅作可视化提示；点击徽标可下钻查看详细解析结果，不触发任何写操作。

7. **导入过程的不可中断**：第三步（导入进度）期间取消按钮隐藏，避免导入到一半被中断导致数据不一致。如确需中止，需通过运维手段处理。

8. **日期/金额格式列用法**：「日期/金额格式」列仅作用于该字段最终值的呈现，对取值逻辑本身无影响。同一字段可独立指定格式（如金额字段填 `#,##0.00`、日期字段填 `yyyy-MM-dd`），无需关心 Excel 单元格样式。常见格式串示例：
   - 金额（千分位 + 两位小数）：`#,##0.00`
   - 金额（仅两位小数）：`0.00`
   - 金额（财务记账风格，负数加括号）：`#,##0.00;(#,##0.00)`
   - 带币种符号：`¥#,##0.00` / `HK$#,##0.00` / `CA$#,##0.00`
   - 日期（ISO）：`yyyy-MM-dd`
   - 日期（中文）：`yyyy年MM月dd日`
   - 月份：`yyyy-MM`
   - 百分比（如税率）：`0.00%`
   - 空值：留空表示不格式化，按原值输出
